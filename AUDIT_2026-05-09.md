## Findings

### High Priority

1. Article content can execute unsafe HTML if an admin or synced source stores hostile markup.
   - Evidence: `src/pages/ArticleDetail.tsx:91` renders `article.content` via `dangerouslySetInnerHTML`.
   - Evidence: `src/components/admin/ArticleManagement.tsx:240` explicitly allows article body HTML.
   - Risk: XSS in the public news page if article content is ever untrusted or copied from a compromised source.
   - Status: Resolved in `src/utils/sanitizeHtml.ts` and `src/pages/ArticleDetail.tsx`.

2. Public Edge Functions using service role need endpoint-by-endpoint authorization review.
   - Evidence: `supabase/config.toml:57-58` makes `create-jt-waybill` public.
   - Evidence: `supabase/functions/create-jt-waybill/index.ts:184-185` accepts direct `{ order_id }` invocation.
   - Risk: a public caller can trigger logistics behavior if the J&T API is enabled and they know or guess an order ID.
   - Status: Resolved for direct invokes and webhooks in `supabase/functions/create-jt-waybill/index.ts`.

3. J&T tracking webhook signature verification is not implemented.
   - Evidence: `supabase/functions/jt-tracking-webhook/index.ts:30`.
   - Risk: forged tracking events could be accepted once update logic is implemented.
   - Status: Resolved in `supabase/functions/jt-tracking-webhook/index.ts`.

### Medium Priority

4. Lint is currently not deploy-gate clean.
   - `npm run lint` reports 31 errors and 15 warnings.
   - Most errors are `no-explicit-any` in admin, checkout, order, hook, page, and Edge Function files.
   - Status: Resolved for lint errors. Warnings remain.

5. Upload proof function does not validate the declared file type and always stores content as JPEG.
   - Evidence: `supabase/functions/upload-order-proof/index.ts:63` uses `contentType: ALLOWED_TYPES[0]`.
   - Risk: poor validation and incorrect metadata for PDFs, PNGs, WebP, and GIFs.
   - Status: Resolved in `supabase/functions/upload-order-proof/index.ts` and `src/pages/UploadProof.tsx`.

6. React test warnings indicate a prop mismatch.
   - Evidence: `src/components/sections/HeroSection.tsx:15` uses `fetchPriority="high"`.
   - Test warning says React does not recognize `fetchPriority`.
   - Status: Resolved in `src/components/sections/HeroSection.tsx`.

7. Checkout relies on the publishable key existing at build time.
   - Evidence: `src/pages/Checkout.tsx:232` reads only `VITE_SUPABASE_PUBLISHABLE_KEY` for direct Edge Function fetch.
   - Client fallback supports multiple env names in `src/integrations/supabase/client.ts`, but Checkout does not.
   - Status: Resolved with shared exports from `src/integrations/supabase/client.ts`.

8. Media validation is not currently useful.
   - `npm run media:validate` passes because `scripts/media-urls.txt` is empty.
   - Risk: broken product/news/storage URLs will not be caught.
   - Fix: generate `scripts/media-urls.txt` from product/article sources or add a DB-backed media audit script.
